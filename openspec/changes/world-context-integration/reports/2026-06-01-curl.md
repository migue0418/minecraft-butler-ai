# Informe pruebas curl — world-context-integration

**Fecha:** 2026-06-01
**Backend:** http://127.0.0.1:8000 (rama feature/world-context-integration)

## 6.3 — /ask sin world_context (retrocompatibilidad)

```
POST /api/butler/ask {"message":"hola Alfred"}
→ 200 OK, type: speak ✓
```

## 6.4 — /ask con world_context, pregunta sobre inventario

```
POST /api/butler/ask {
  "message": "tengo hierro?",
  "world_context": {"player": {"inventory": [{"item":"minecraft:iron_ingot","count":5}], ...}, ...}
}
→ 200 OK, type: speak
→ "¡Sí! Tienes 5 lingotes de hierro (iron_ingot) en tu inventario..."
```
Alfred usó los datos del inventario. needs_world_context=True activado. ✓

## 6.5 — /ask con world_context, pregunta de crafteo (no debe usar contexto)

```
POST /api/butler/ask {"message":"como crafteo una espada de diamante", "world_context": {...}}
→ 200 OK, type: speak
→ "## Crafteo de Espada de Diamante — Para craftear una espada de diamante necesitas..."
```
Respuesta genérica sin mencionar inventario. needs_world_context=False. ✓

## 6.6 — /ask-voice con world_context válido (multipart)

```
POST /api/butler/ask-voice (audio=silence.wav, world_context={"player":...})
→ 422 "No se pudo transcribir el audio."
```
El form field world_context fue parseado correctamente (error 422 por audio silencioso, no por world_context). ✓

## 6.7 — /ask-voice con world_context JSON malformado

```
POST /api/butler/ask-voice (audio=silence.wav, world_context="{broken json here}")
→ 422 (por audio silencioso, no 500)
```
JSON malformado ignorado silenciosamente con log de warning. ✓

## Resultado

Todos los escenarios funcionan correctamente. La integración es retrocompatible.
