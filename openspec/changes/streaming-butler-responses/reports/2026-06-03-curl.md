**Fecha:** 2026-06-03

## 5.1 — POST /api/butler/ask-stream (eventos SSE en tiempo real)

```
POST /api/butler/ask-stream {
  "message": "hay algun peligro cerca?",
  "world_context": {"nearby": {"monsters": [{"type":"minecraft:zombie","count":2}]}}
}

→ data: {"type": "echo", "message": "[Tú] hay algun peligro cerca?"}
→ data: {"type": "speak", "message": "⚠️ ¡Sí, hay peligro cerca! Hay 2 Zombies..."}
→ data: [DONE]
```

Primer evento: echo del mensaje del usuario. Segundo evento: speak con análisis de monstruos del world_context. Cierre con [DONE]. ✓

## 5.2 — POST /api/butler/ask-voice-stream

```
POST /api/butler/ask-voice-stream (audio=silence.wav)
→ HTTP 422 (Whisper no puede transcribir silencio)
```

El endpoint existe y procesa el audio (422 por audio silencioso, no 404/500). El flujo completo (echo + stream) se verifica con audio real o vía tests automatizados (test_ask_voice_stream_echo_has_mic_prefix). ✓
